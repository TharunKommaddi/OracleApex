**Process Overview:**
The process is designed to automatically set or update the end dates (expiration dates) for predecessor regulations when a new regulation is introduced or an existing regulation is modified. This is achieved through the `PCK_RI_HISTORY` package, which contains two procedures: `psetEndDateVorgaenger` and `pSetEndDateVorschrift`.

**Procedure: `psetEndDateVorgaenger`**
- **Purpose:** This procedure calculates and sets the end date for predecessor regulations based on the given regulation ID and the introduction of new effective dates or changes to existing ones.
- **Parameters:**
  - `avorschriftid` (NUMBER): The ID of the current regulation.
  - `anotvorschriftid` (NUMBER, optional): The ID of a regulation to be excluded from the predecessor calculation.
- **Process Flow:**
  1. Retrieves all predecessor regulations (`Vorgänger`) of the current regulation from the `t_vorschrift_ref_vorschrift` table, excluding the regulation specified in `anotvorschriftid`.
  2. Joins with the `t_vorschrift_ref_einsatzdatum_typ` table to retrieve effective dates (`einsatzdatum`) for different vehicle types (`einsatzdatum_typid`) for both predecessor and successor regulations.
  3. Calculates the minimum effective date for each predecessor regulation, considering different date types (`Datum_type`).
  4. Updates or inserts records in the `t_vorschrift_ref_einsatzdatum_typ` table with the calculated end date for each predecessor regulation.

**Procedure: `pSetEndDateVorschrift`**
- **Purpose:** This procedure calls the `psetEndDateVorgaenger` procedure for the given regulation ID and all its successor regulations.
- **Parameters:**
  - `avorschriftid` (NUMBER): The ID of the current regulation.
- **Process Flow:**
  1. Calls the `psetEndDateVorgaenger` procedure for the current regulation (`avorschriftid`), excluding itself from the predecessor calculation for its successors.
  2. Calls the `psetEndDateVorgaenger` procedure for all successor regulations of the current regulation, excluding the current regulation from the predecessor calculation.

**Data Structures Involved:**
- `t_vorschrift_ref_vorschrift`: Table containing information about predecessor-successor relationships between regulations.
- `t_vorschrift_ref_einsatzdatum_typ`: Table containing effective dates (`einsatzdatum`) for different vehicle types (`einsatzdatum_typid`) for each regulation.

**Calculation of End Dates:**
The process calculates the end dates for predecessor regulations based on the minimum effective date of the successor regulation, considering different date types (`Datum_type`). The date types are defined as follows:
- 1060:  Effective date for `Datum Außer Kraft FBU` vehicle types (related to `einsatzdatum_typid` 1023).
- 1063: Effective date for `Datum Außer Kraft CKD` vehicle types (related to `einsatzdatum_typid` 1021).
- 1041: Effective date for `Außerkraftsetzungsdatum` vehicle types.

**Updates and Insertions:**
The calculated end date for each predecessor regulation is either updated in an existing record or inserted as a new record in the `t_vorschrift_ref_einsatzdatum_typ` table, along with a description ('System Berechnung über Nachfolger') and a manual entry flag (`MANUELLE_EINTRAGUNG`) set to 20.
