# PCK_RI_HISTORY Package

The `PCK_RI_HISTORY` package is a PL/SQL package that contains two procedures:

## psetEndDateVorgaenger

```sql
procedure psetEndDateVorgaenger(avorschriftid number, anotvorschriftid number default 0);
```

This procedure is responsible for setting the end date for predecessor regulations (Vorschriften) based on the provided successor regulation (`avorschriftid`) and an optional exclusion regulation (`anotvorschriftid`).

### Steps:

1. Retrieve all predecessor regulations for the given successor regulation (`avorschriftid`) from the `t_vorschrift_ref_vorschrift` table, excluding the `anotvorschriftid` if provided.
2. Fetch the deployment dates (`einsatzdatum` and `neue_typen`) for the successor regulation and its predecessors from the `t_vorschrift_ref_einsatzdatum_typ` table.
3. Calculate the minimum deployment date for each predecessor regulation, considering different types of deployment dates (`Datum_type`).
4. For each predecessor regulation:
   - Determine the end date (`einsatzdatum`) as one day before the minimum deployment date.
   - Insert or update the end date and related information in the `t_vorschrift_ref_einsatzdatum_typ` table.

## pSetEndDateVorschrift

```sql
procedure pSetEndDateVorschrift(avorschriftid number);
```

This procedure is a wrapper around `psetEndDateVorgaenger`. It iterates over the provided regulation (`avorschriftid`) and its successors, calling `psetEndDateVorgaenger` for each regulation, excluding the `avorschriftid` itself.

### Steps:

1. Retrieve the `avorschriftid` and its successor regulations from the `t_vorschrift_ref_vorschrift` table.
2. For each regulation:
   - Call `psetEndDateVorgaenger` with the current regulation ID and the `avorschriftid` as the exclusion regulation.
